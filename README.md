# 화면설계서 AI 어시스턴트

서비스기획자(UX/PM)를 위한, **서버 설치 없이 브라우저에서 바로 쓰는** 화면설계서 도우미입니다.

- **패턴 컨설턴트**: 채팅으로 UI 패턴 고민(FAQ 아코디언 vs 별도 페이지, 필터 배치, 폼 유효성 검사 타이밍 등)을 물어보면 내장 룰셋 + 필요 시 웹 검색으로 답변하고, 새로 찾은 지식은 확인 후 룰셋에 저장해 다음부터 재사용합니다.
- **QA 리뷰어**: 화면설계서(.pptx) 또는 **피그마 파일 URL**을 업로드/입력하면 도형(노드) 좌표를 분석해 정렬/간격/폰트 일관성을 점검하고, Claude API로 맞춤법을 검토합니다. PPTX는 슬라이드를 PNG로 내보내 함께 업로드하면 여백·시각적 위계도 추가로 분석하고, 피그마는 컴포넌트 인스턴스 간 스타일 불일치를 자동 탐지합니다.
- **레퍼런스 디자인**: 룰셋에 텍스트 규칙뿐 아니라 "이미 승인된 화면" 자체를 기준으로 등록할 수 있습니다(PPTX 슬라이드 또는 피그마 프레임 선택). QA 리뷰 시 같은 카테고리의 레퍼런스와 자동 비교해 "폰트 크기가 14px→16px로 다릅니다" 같은 구체적 차이를 리포트합니다.

## 1. 사용 방법

1. `index.html` 파일을 더블클릭해서 브라우저(Chrome 권장)로 엽니다. **npm install이나 서버 실행이 필요 없습니다.**
2. 상단의 "키 발급받기" 링크로 이동해 [Anthropic Console](https://console.anthropic.com/settings/keys)에서 API 키를 발급받습니다. (Anthropic 계정 및 결제 정보 등록이 필요합니다.)
3. 발급받은 키(`sk-ant-...`)를 상단 입력창에 붙여넣습니다. 이후 브라우저에는 계속 저장되어 있으므로 매번 입력할 필요가 없습니다.
4. 좌측 채팅창에 UI 패턴 고민을 입력하거나, 우측 "QA 리뷰어" 탭에서 pptx 파일을 업로드하거나 피그마 URL을 입력해 사용합니다.
5. 피그마 연동을 쓰려면 설정 탭에서 Figma Personal Access Token(Figma 계정 설정 > Security 탭에서 발급)도 입력해주세요.

> API 키는 이 브라우저의 `localStorage`에만 저장되며 외부로 전송되지 않습니다(Anthropic API 서버 요청 시에만 사용). 개인용 로컬 도구이므로 이 정도 보안 수준으로 설계했습니다. 여러 사람이 함께 쓰는 PC라면 사용 후 설정 탭에서 "전체 삭제"를 눌러주세요.

## 2. 주요 기능

### 패턴 컨설턴트
- 질문을 입력하면 먼저 내장 룰셋(초기 약 40개, Nielsen Norman Group / Baymard Institute / Material Design 3 / Apple HIG 기반)에서 관련 항목을 찾아 답합니다.
- 룰셋에 근거가 부족하면 Claude의 웹 검색 기능으로 신뢰도 높은 출처를 조사한 뒤 답합니다. 이 경우 답변에 "🔎 웹 검색으로 보강됨" 뱃지가 붙습니다.
- 웹 검색으로 새로운 지식을 얻으면 채팅창에 "이 답변을 룰셋에 저장할까요?" 카드가 뜹니다. **자동 저장되지 않으며**, 저장 버튼을 눌러야 룰셋에 추가됩니다.

### QA 리뷰어
상단에서 **PPTX 업로드** 또는 **피그마 URL로 검토** 중 하나를 선택합니다.

**PPTX**
1. `.pptx` 파일을 업로드합니다(필수). 브라우저에서 JSZip으로 압축을 풀어 슬라이드 XML을 직접 파싱하므로 파일이 외부로 업로드되지 않습니다(맞춤법 검토를 위해 추출된 텍스트만 Claude API로 전송됩니다).
2. (선택) 파워포인트에서 슬라이드를 PNG로 내보내 업로드하면, 좌표만으로는 판단하기 어려운 여백·정렬감·시각적 위계를 Claude Vision으로 추가 분석합니다. 각 이미지가 몇 번 슬라이드인지 지정해주세요.

**피그마**
1. 피그마 파일 URL(`https://www.figma.com/design/...`) 또는 파일 key를 입력합니다.
2. (선택) 프레임 이름 또는 node-id로 필터링할 수 있습니다. 비워두면 각 페이지의 최상위 프레임을 최대 20개까지 검토합니다.
3. 브라우저에서 Figma REST API(`GET /v1/files/:key`)를 직접 호출합니다. Figma API는 `Access-Control-Allow-Origin: *`로 CORS를 자체 허용하므로 `file://`로 열어도, 별도 서버·프록시 없이 정상 동작합니다.
4. 컴포넌트 인스턴스(`componentId`) 정보가 있으므로, 같은 컴포넌트인데 폰트 크기/색상이 다르거나 마스터 대비 속성을 오버라이드한 인스턴스를 자동으로 탐지해 리포트에 포함합니다.

**공통**
- 룰셋에 저장된 카테고리를 하나 선택하면, 같은 카테고리의 "레퍼런스 디자인"과 자동으로 비교해 구체적인 차이(예: 폰트 크기 14px→16px)를 리포트에 추가합니다.
- "QA 리뷰 실행"을 누르면 슬라이드/프레임별로 오류(빨강)/권장(주황)/제안(파랑) 등급으로 이슈를 정리해 보여줍니다.

좌표 기반 규칙 검사는 자동으로 다음을 점검합니다: 경계 이탈, 동일 라인 요소의 y좌표 불일치, 반복 요소(카드/버튼 등) 간 간격 불균형, 폰트 크기/종류 과다 사용.

### 룰셋 뷰어
- 카테고리별로 룰셋을 확인·검색·편집·삭제할 수 있습니다.
- 두 가지 타입이 있으며 시각적으로 구분됩니다:
  - **텍스트 규칙** (`type: "rule"`): 조건/추천/근거 등 텍스트 기반 원칙. "+ 텍스트 규칙" 버튼으로 직접 추가하거나 패턴 컨설턴트 채팅에서 저장됩니다.
  - **레퍼런스 디자인** (`type: "reference_design"`, 🖼️ 뱃지로 표시): "이미 승인된 화면" 자체를 기준으로 등록. "+ 레퍼런스 디자인" 버튼을 누르면 PPTX 슬라이드 또는 피그마 프레임을 선택하고 카테고리/태그/메모를 입력해 저장할 수 있습니다. 저장된 좌표·텍스트·폰트 데이터는 QA 리뷰어의 일관성 비교에 사용됩니다.
- JSON으로 내보내기/가져오기가 가능해 직접 검토·수정하거나 백업할 수 있습니다.
- 설정 탭에서 언제든 초기 시드 데이터로 되돌릴 수 있습니다.

## 3. 파일 구성

```
index.html        메인 화면 (더블클릭으로 실행)
style.css         스타일
app.js            앱 로직 (API 호출, PPTX/피그마 파싱, QA 검사, 룰셋 관리 등)
ruleset-seed.js   초기 시드 룰셋 데이터 (최초 실행 시 1회 localStorage로 복사됨)
```

## 4. 기술 메모

- 서버/백엔드 없음. 정적 파일만으로 동작하며 `file://`로 직접 열어도 됩니다.
- Claude API는 브라우저에서 `fetch()`로 직접 호출하며, `anthropic-dangerous-direct-browser-access: true` 헤더로 브라우저 직접 호출을 허용합니다. API 키가 클라이언트 코드에 노출되므로 **개인 로컬 사용 전용**입니다. 여러 사용자가 접근 가능한 서버에 이 파일을 그대로 배포하지 마세요.
- Figma REST API도 같은 이유로 브라우저에서 직접 호출합니다. Figma API는 자체적으로 `Access-Control-Allow-Origin: *`를 응답하므로(직접 curl로 확인) 프록시나 로컬 서버가 필요 없습니다. Figma 토큰도 localStorage에만 저장됩니다.
- `.pptx`는 사실 ZIP 파일이라는 점을 이용해 JSZip(CDN)으로 열고, 내부 슬라이드 XML(`ppt/slides/slideN.xml`)을 `DOMParser`로 파싱해 도형 좌표(EMU 단위를 px로 환산)·텍스트·폰트 정보를 추출합니다.
- 피그마는 파일의 노드 트리(JSON)를 그대로 받아와 재귀 순회하며, 프레임 좌상단을 원점으로 하는 상대좌표로 환산해 PPTX와 동일한 shape 스키마로 변환합니다. 이 덕분에 정렬/간격/폰트 검사 로직을 소스에 상관없이 그대로 재사용합니다. `INSTANCE` 노드의 `componentId`/`overrides`를 이용해 컴포넌트 일관성(같은 컴포넌트인데 스타일이 다른 인스턴스)을 탐지합니다.
- 레퍼런스 디자인과의 비교는 좌표를 규칙 기반으로 1:1 매칭하는 대신, 두 화면의 요소 목록(JSON)을 Claude에 함께 전달해 의미 있는 차이만 구조화된 형태로 뽑아내는 방식입니다(기존 맞춤법/Vision 검사와 동일한 패턴).
- 슬라이드 이미지를 브라우저에서 직접 렌더링하는 기능은 없습니다(파워포인트→PPT 렌더링은 브라우저 JS만으로 구현하기 어려움). 대신 사용자가 파워포인트에서 PNG로 내보내 업로드하면 Claude Vision으로 분석합니다.
- 데이터 저장은 전부 `localStorage` + JSON export/import로 처리하며 별도 DB가 없습니다.

## 5. 향후 확장 (이번 스프린트 제외)

- 컴포넌트 일관성 체커 고도화 (현재는 인스턴스 간 폰트/색상/오버라이드 비교 정도의 최소 버전만 지원)
- 화면설계서 초안 자동 생성
